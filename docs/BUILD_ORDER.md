# SceneSmith Build Order

SceneSmith is built documentation-first.

No subsystem should be implemented until its purpose, authority, boundaries, failure modes, and tests are understood.

## Phase 1: Project Foundation

* Repository structure
* Development rules
* Architecture outline
* Python project configuration
* Test package scaffold

## Phase 2: Core Domain Model

* Story
* Chapter
* Scene
* Evidence
* Extracted fact
* Canon state

## Phase 3: Story Import System

* Plain text import
* Markdown import
* Chapter splitting
* Scene splitting
* Source order preservation

## Phase 4: Canon Engine

* Canon state storage
* Evidence-backed updates
* Unknown state handling
* Confidence tracking

## Phase 5: Character Engine

* Character identity
* Appearance state
* Equipment state
* Relationship state
* Character history

## Phase 6: Timeline Engine

* Chapter order
* Scene order
* Event order
* State reconstruction

## Phase 7: Scene Engine

* Characters present
* Location context
* Time and weather context
* Active relationships
* Current world state

## Phase 8: Prompt Engine

* Image prompts
* Narration prompts
* Camera prompts
* Animation prompts

## Phase 9: Export Engine

* JSON export
* Markdown export
* CSV export
* Character sheets
* Scene sheets
* Prompt sheets
